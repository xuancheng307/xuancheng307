# `skills/` — 技能盤點

分三個級別：advanced / intermediate / learning。每個 .md 一個 list，一行一個技能。

## 維護原則

1. **「精通」用 advanced，不要寫精通中文** — 跟英文版對齊
2. **誠實** — 沒做過 production 的東西不放 advanced
3. **跟 entry 連動** — 主要技能應該在 projects/research/experience 至少出現過一次
4. **每月或每季 review** — 新學的 → learning；用熟了 → intermediate；有 production track record → advanced

## 三級定義

- **Advanced**：能設計 + 實作 + debug 邊界問題（framework patch level）
- **Intermediate**：能用、能修，但設計重大架構會猶豫
- **Learning**：用過但沒 production，或還在學

## 同類技能 cluster

避免列 10 個小東西 — 用 cluster：
- ✅「MySQL / PostgreSQL · Materialized View · Schema Design」
- ❌「MySQL · PostgreSQL · JOIN · GROUP BY · CONCURRENTLY REFRESH · ...」
